01 — 서비스 개요

AI 스마트 비서는 사용자의 업무를 요청 기반 → 제안 기반 → 자동 수행까지 단계적으로 확장하며 업무를 대신 수행하는 능동형 AI 에이전트이다. 세 가지 핵심 기능이 유기적으로 연결되어 브리핑 → 실행 → 자동화의 순환 구조를 형성한다.

개발 우선순위

P1 — 사용자 요청 기반 (Reactive AI) P2 — AI 선제 제안 + 사용자 검수 (Assistive AI) P3 — AI 자동 수행 (Autonomous AI)

데이터 레벨 정의

레벨구분설명우선순위
L1구조화 데이터업무, 일정, 담당자, 상태 등 명확한 데이터P1부터
L2반구조화 데이터댓글, 파일, 활동 로그 등 연관 정보P2부터
L3비정형 데이터채팅, 자연어, 맥락 기반 데이터P3부터

02 — 일일 업무 브리핑 (3-1)

AI가 사용자의 업무 데이터를 분석하여 하루 업무 현황, 우선순위, 리스크를 요약 제공하는 기능이다. 브리핑이 단순 정보 전달로 끝나지 않고, 후속 액션 Quick Prompt를 통해 3-2 AI 업무 직접 요청 기능으로 자연스럽게 연결되는 것이 핵심 가치이다.

03 — AI 업무 직접 요청 (3-2)

사용자가 자연어로 업무를 요청하면 AI가 업무 생성, 수행, 완료까지 담당하는 기능이다. 핵심은 대화형 루프 구조로, AI가 중간중간 사용자와 주고받으며 결과물을 점진적으로 완성해 나간다. 사용자 확인 단계에서 승인/수정/추가 액션으로 분기되는 구조가 특징이다.

04 — AI 리마인더 (3-3)

사용자의 업무, 일정, 활동 데이터를 기반으로 특정 시점 또는 조건에서 알림을 제공하거나 AI 실행을 통해 필요한 작업을 수행하는 기능이다. 단순 이벤트 알림을 넘어, 상황을 판단하고 행동을 유도하거나 자동 실행까지 수행하는 비서형 리마인더를 제공하는 것을 목표로 한다. 리마인더는 전체 서비스의 자동 순환 엔진 역할을 한다.

05 — 핵심 가치 및 향후 확장

핵심 가치

업무 자동화 단계 확장 — 요청 → 제안 → 자동 수행
데이터 기반 의사결정 — 플로우 + 메일 + 일정 통합 분석
실행 중심 AI — 단순 답변이 아니라 업무를 직접 수행
자연어 인터페이스 — 사용자는 "말만 하면 됨"

향후 확장: 비서 자동 응답 (Agent Proxy Response)

사용자가 부재중이거나 응답이 지연될 경우, AI 비서가 사용자를 대신하여 메시지에 응답하는 기능으로, P3 자동 수행의 연장선에서 개발 예정이다.
