# FLOW-001 브리핑 데이터 수집·분류 스펙

---

## 1. 문서 목적

오전 브리핑(3개 섹션)과 퇴근 마무리 리포트의 데이터 수집 범위, 필터 조건, 섹션 간 중복 판단 규칙, LLM 전달 형식을 정의합니다.

---

## 2. 오전 브리핑 — 섹션 구성

오전 브리핑은 3개 섹션 + 추천 순서로 구성됩니다.

| 순서 | 섹션명 | 데이터 성격 |
|------|--------|------------|
| 1 | 오늘의 일정 | 당일 캘린더 이벤트 |
| 2 | 마감 업무 | 당일·익일 마감 업무 |
| 3 | 퇴근 후 변화 | 부재중 발생한 변경사항 |
| — | 오늘의 추천 순서 | 위 3개 섹션 종합 분석 (LLM 생성) |

---

## 3. 섹션별 판단 범위

### 3-1. 오늘의 일정

**수집 대상**
- Flow 캘린더에서 당일(00:00 ~ 23:59) 시작하는 이벤트
- 나(사용자)가 참석자로 등록된 일정만 포함

**필터 조건**

| 조건 | 포함/제외 |
|------|----------|
| 내가 참석자인 일정 | 포함 |
| 종일 일정 | 포함 (별도 표시: "종일") |
| 취소된 일정 | 제외 |
| 거절한 일정 | 제외 |
| 반복 일정 (당일 인스턴스) | 포함 |

**정렬**: 시작 시간 오름차순
**표시 건수**: 최대 4건, 초과 시 "+N건 더" 표시

**외부 미팅 판단**
- 참석자 중 외부 도메인 또는 `(회사명)` 형태가 1명 이상 → `external: true`
- 외부 미팅 시 회사명 표시 (예: "넥스트랩 AI 솔루션 도입 미팅 🏢")

**LLM에 전달하는 필드**

| 필드 | 타입 | 설명 |
|------|------|------|
| id | string | 일정 고유 ID |
| time | string | 시작 시각 (HH:mm) |
| title | string | 일정명 |
| location | string | 장소 (없으면 빈 문자열) |
| attendees | string[] | 참석자 이름 목록 |
| external | boolean | 외부 미팅 여부 |
| company | string? | 외부 회사명 (external=true일 때) |

---

### 3-2. 마감 업무

업무는 사용자의 역할에 따라 두 가지로 구분됩니다.

| 구분 | 조건 | 성격 |
|------|------|------|
| **내가 해야할 업무** | 사용자가 담당자인 업무 | 직접 처리·실행 |
| **내가 확인해야할 업무** | 사용자가 작성자이고 다른 사람이 담당자인 업무 | 진행 상황 추적·확인 |

#### 내가 해야할 업무

**수집 대상**
- Flow 업무 중 내가 **담당자**인 업무
- 마감일이 **당일(D+0)** 또는 **익일(D+1)**인 업무

**필터 조건**

| 조건 | 포함/제외 |
|------|----------|
| 담당자가 나인 업무 | 포함 |
| 상태: 대기, 진행중 | 포함 |
| 상태: 완료, 보류, 취소 | 제외 |
| 상위업무 (하위업무 있음) | 포함 |
| 하위업무 | 포함 (경로에 상위 계층 표시) |
| 마감일 미설정 | 제외 |

**정렬**: 우선순위 내림차순 → 마감일 오름차순

| 우선순위 | 정렬 순서 | 배지 |
|---------|----------|------|
| 긴급 (URGENT) | 1 | 🚨 빨강 |
| 높음 (HIGH) | 2 | 🟠 주황 |
| 보통 (NORMAL) | 3 | 🟢 초록 |
| 낮음 (LOW) | 4 | ➖ 파랑 |

**표시 건수**: 최대 5건

#### 내가 확인해야할 업무

**수집 대상**
- Flow 업무 중 내가 **작성자**이고 다른 사람이 **담당자**인 업무
- 마감일이 **당일(D+0)** 또는 **익일(D+1)**인 업무

**필터 조건**

| 조건 | 포함/제외 |
|------|----------|
| 작성자가 나이고 담당자가 다른 사람인 업무 | 포함 |
| 상태: 대기, 진행중 | 포함 |
| 상태: 완료, 보류, 취소 | 제외 |
| 마감일 미설정 | 제외 |

**정렬**: 마감일 오름차순 → 상태 '요청' 우선 (미착수 = 확인 필요)
**표시 건수**: 최대 5건

#### LLM에 전달하는 필드 (공통)

| 필드 | 타입 | 설명 |
|------|------|------|
| id | string | 업무 고유 ID |
| title | string | 업무명 |
| due | string | 마감 표시 ("오늘 마감" / "내일 마감") |
| priority | enum | URGENT / HIGH / NORMAL / LOW |
| status | string | 현재 상태 (대기 / 진행중) |
| role | enum | assignee(담당자) / requester(작성자) |
| assignee | string? | 담당자 이름 (role=requester일 때) |
| path | string[] | 계층 경로 (프로젝트 > 상위업무 > 업무명) |

---

### 3-3. 퇴근 후 변화

**수집 대상**
- 사용자의 **전날 마지막 접속 시각** ~ **브리핑 생성 시각** 사이에 발생한 변경

**수집 범위 (변경 유형)**

| 유형 | 조건 | 아이콘 |
|------|------|--------|
| 신규 업무 배정 (new_task) | 내가 담당자로 새로 배정된 업무 | 📌 |
| 상태 변경 (status) | 내 업무(담당/참여)의 상태·우선순위 변경 | 🔄 |
| 멘션 (mention) | 나를 멘션한 댓글·대댓글 (글·업무·일정 게시물) | 💬 |
| 댓글 (comment) | 내가 작성한 게시물에 달린 댓글 | 💬 |
| 대댓글 (reply) | 내가 작성한 댓글에 달린 대댓글 | 💬 |

**필터 조건**

| 조건 | 포함/제외 |
|------|----------|
| 위 5가지 유형에 해당 | 포함 |
| 내가 직접 수행한 변경 | 제외 |
| 시스템 자동 변경 (봇, 워크플로우) | 제외 |

**정렬**: 발생 시각 역순 (최신순)
**표시 건수**: 최대 5건 기본 표시, 초과 시 "+N건 더 보기" 접기/펼치기

**LLM에 전달하는 필드**

| 필드 | 타입 | 설명 |
|------|------|------|
| id | string | 변경 고유 ID |
| type | enum | new_task / status / mention / comment / reply |
| title | string | 관련 게시물·업무 제목 |
| from | string | 변경을 발생시킨 사람 |
| timeAgo | string | 경과 시간 ("3시간 전") |
| path | string[]? | 계층 경로 (있는 경우) |
| change | string? | 상태 변경 내용 (status 유형만, 예: "진행중 → 완료") |
| preview | string? | 댓글/멘션 원문 미리보기 (mention 유형만) |
| chatRoom | string? | 채팅방 출처 (채팅 멘션일 경우) |

## 4. 섹션 간 중복 판단 규칙

동일 항목이 복수 섹션에 해당할 경우 아래 규칙으로 처리합니다.

### 4-1. 마감 업무 ↔ 퇴근 후 변화

| 상황 | 처리 |
|------|------|
| 마감 업무에 표시된 업무가 퇴근 후 상태 변경됨 | **양쪽 모두 표시**. 마감 업무 섹션에는 현재 상태 기준, 퇴근 후 변화에는 변경 이력 표시 |
| 마감 업무가 퇴근 후 "완료"로 변경됨 | 마감 업무에서 **제외** (완료 상태 필터), 퇴근 후 변화에만 표시 |
| 퇴근 후 새로 배정된 업무가 오늘 마감 | **양쪽 모두 표시**. 퇴근 후 변화에 "신규 배정" 표시, 마감 업무에 마감 표시 |

### 4-2. 오늘의 일정 ↔ 퇴근 후 변화

| 상황 | 처리 |
|------|------|
| 퇴근 후 일정 관련 멘션·댓글 발생 | 일정 섹션에는 일정 정보만 표시, 퇴근 후 변화에 멘션·댓글 표시 (중복 허용) |
| 퇴근 후 일정이 취소됨 | 오늘의 일정에서 **제외** (취소 필터), 퇴근 후 변화에 취소 변경 표시 |

### 4-3. 일반 원칙

- **같은 항목이라도 섹션 관점이 다르면 중복 표시 허용** (마감 업무 → "마감 임박" 관점, 퇴근 후 변화 → "상태 변경" 관점)
- LLM 요약에서는 중복 참조 시 "아까 말씀드린 ~" 형태로 자연스럽게 연결
- 추천 순서(마지막 영역)에서 모든 섹션을 종합하여 우선순위 정리

## 5. 퇴근 마무리 리포트 — 데이터 범위

### 5-1. 수집 시점
- 트리거: 18:00 이후 + 마지막 Flow 접속으로부터 1시간 경과
- 수집 범위: 당일 업무 현황 + 익일 일정

### 5-2. 섹션 구성

| 순서 | 섹션명 | 데이터 범위 |
|------|--------|------------|
| 1 | 오늘 업무 현황 | 당일 처리 완료 + 미처리 업무 |
| 2 | 오늘 하루 분석 | 일정 참석 건수 + 업무 완료 건수 종합 |
| 3 | 미회신 알림 | 미회신 멘션·댓글 중 처리 안 된 항목 |
| 4 | 내일 일정 미리보기 | 익일 캘린더 이벤트 |

### 5-3. 오늘 업무 현황

**수집 대상**
- 당일 브리핑에 포함되었던 마감 업무
- 당일 중 상태가 "완료"로 변경된 업무
- 당일 퇴근 후 변화에서 신규 배정된 업무

**판단 기준**

| 상태 | 표시 |
|------|------|
| 업무 상태가 "완료" | ✅ 완료 + 완료 시각 또는 메모 |
| 업무 상태가 대기·진행 | ⏳ 미처리 + 추천 처리 시점 |

**정렬**: 완료 업무 먼저 → 미처리 업무 (우선순위 내림차순)

### 5-4. 미회신 알림

**수집 대상**
- 오전 브리핑 "퇴근 후 변화"에서 mention 유형 항목 중, 퇴근 시점까지 회신하지 않은 건

**판단 기준**
- 해당 멘션의 게시물·댓글에 사용자의 후속 댓글이 없으면 "미회신"

### 5-5. 내일 일정 미리보기

**수집 대상**
- 오전 브리핑 "오늘의 일정"과 동일한 로직, 날짜만 익일(D+1) 적용
- 필터 조건, 정렬, 외부 미팅 판단 모두 동일

## 6. 데이터 수집 타이밍

### 오전 브리핑

| 단계 | 시점 | 설명 |
|------|------|------|
| 수집 시작 | 알림 발송 시각 **30분 전** | 예: 08:00 알림 → 07:30부터 수집 |
| 수집 완료 | 알림 발송 시각 **직전** | 최신 데이터 반영 |
| LLM 생성 | 수집 완료 즉시 | 4개 섹션 + 추천 순서 생성 |
| 세션 생성 + Push | 알림 발송 시각 | 설정된 시각에 발송 |

### 퇴근 마무리 리포트

| 단계 | 시점 | 설명 |
|------|------|------|
| 트리거 감지 | 18:00 이후, 마지막 접속 + 1시간 경과 | 서버 측 접속 로그 기반 |
| 수집 | 트리거 발생 즉시 | 당일 업무 현황 + 익일 일정 수집 |
| LLM 생성 | 수집 완료 즉시 | 리포트 생성 |
| 세션 생성 + Push | LLM 생성 완료 후 | 즉시 발송 |

### 설정 화면 힌트 텍스트

| 알림 유형 | 힌트 |
|----------|------|
| 오전 브리핑 | "📡 {시각} 30분 전부터 일정·업무·메시지를 수집해요" |
| 퇴근 마무리 | "📡 퇴근 시점의 업무 현황·내일 일정을 수집해요" |

## 7. LLM 프롬프트 전달 형식

### 오전 브리핑 프롬프트 구조

```
[시스템 프롬프트]
- 역할: AI 비서 {캐릭터 이름}, {성격} 톤
- 규칙: 한국어, 존댓말, 1~2문장 핵심 전달
- Prompt Guard 정책 적용

[데이터 컨텍스트] (사용자에게 비표시)
- schedules: [{일정 데이터}]
- tasks: [{업무 데이터}]
- absence: { summary, items: [{변화 데이터}] }

[생성 지시]
1. 시간대별 인사
2. 섹션 1: 일정 분석 (시간 흐름 기반 인사이트)
3. 섹션 2: 마감 업무 분석 (긴급도 기반 우선순위 제안)
4. 섹션 3: 퇴근 후 변화 요약 (액션 필요 항목 강조)
5. 추천 순서: 위 3개 섹션 종합하여 효율적 처리 순서 제안
```

### LLM 생성 규칙

| 규칙 | 설명 |
|------|------|
| 댓글 원문 비노출 | 댓글은 요약 형태로만 제공, 원문 그대로 노출 금지 |
| Prompt Guard 마스킹 | 민감 정보 감지 시 마스킹 처리 후 요약, 뱃지 표시 |
| 섹션 간 자연 연결 | 중복 항목 참조 시 "아까 말씀드린 ~" 형태로 연결 |
| 빈 섹션 처리 | 해당 섹션 데이터 없으면 빈 상태 메시지 표시 후 다음 섹션으로 |

## 8. 빈 상태 조건 정리

| 섹션 | 빈 상태 조건 | 빈 상태 메시지 |
|------|-------------|---------------|
| 오늘의 일정 | 당일 참석 일정 0건 | "오늘 예정된 일정이 없어요" |
| 마감 업무 | 당일·익일 마감 업무 0건 | "오늘과 내일 마감인 업무가 없어요" |
| 퇴근 후 변화 | 마지막 접속 이후 변경 0건 | "놓친 변화가 없어요" |
| 브리핑 전체 | 3개 섹션 모두 데이터 0건 | 브리핑 미발송, 알림 없음 |
| 퇴근 리포트 — 업무 현황 | 당일 관련 업무 0건 | "오늘은 업무 변동이 없었어요" |
| 퇴근 리포트 — 내일 일정 | 익일 참석 일정 0건 | "내일은 예정된 일정이 없어요" |

## 9. 오픈 이슈
