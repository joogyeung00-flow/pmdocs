## 1. 개요

본 문서는 플로우 AI 비서의 브리핑 생성 및 동작에 대한 기준을 정의한다.  
브리핑은 단순 정보 제공이 아닌, 사용자의 업무 상황을 분석하고 행동을 유도하는 것을 목표로 한다.

---

## 2. 브리핑 생성 정책


### 2-1. 생성 트리거

브리핑은 생성 이유에 따라 다음과 같이 구분된다.

#### 1. 예약 실행형 (Scheduled)
사용자가 설정한 시간 또는 시스템에 정의된 시점에 자동 실행되는 브리핑

- 오전 브리핑
- 일정 시작 전 브리핑
- 업무 마감 전 브리핑
- 사용자 설정 브리핑

#### 2. 변화 감지형 (Change-driven)
업무, 메시지, 일정 등 주요 데이터의 변화가 발생했을 때  
사용자에게 의미 있는 변화로 판단되는 경우에만 실행되는 브리핑

- 새 업무 할당
- 멘션 발생
- 업무 상태 변경
- 일정 변경

#### 3. 상태 판단형 (State-based)
사용자의 활동 상태 또는 업무 흐름 상태를 기준으로 실행되는 브리핑

- 마무리 브리핑: 퇴근 시간 이후 또는 마지막 접속 1시간 경과 시 실행
- 장시간 미처리 상태

#### 4. 분석 예측형 (Analytical / Predictive)
데이터를 분석하여 리스크나 병목 가능성이 높다고 판단될 때 실행되는 브리핑

- 마감 임박 + 진행률 부족
- 일정 충돌
- 업무 병목 발생
- 우선순위 충돌

### 2-2. 생성 원칙

- 브리핑은 트리거 발생 시 독립적으로 생성된다.
- 동일 시간대에 여러 브리핑이 생성될 수 있다.
- 단, 콘텐츠는 중복되지 않도록 제어한다.

---

### 2-3. 브리핑 생성 방식

브리핑은 유형에 따라 서로 다른 생성 방식을 가진다.

#### 1. 사전 준비형

발송 시각이 정해진 브리핑은 사용자에게 즉시 제공될 수 있도록  
사전에 데이터 수집 및 응답 생성을 시작한다.

- 오전 브리핑
- 일정 브리핑
- 예약형 사용자 설정 브리핑

기본 기준:
- 발송 시각 기준 30분 전부터 생성 시작

예:
- 브리핑 발송 시각: 08:00  
- 생성 시작 시각: 07:30  

---

#### 2. 즉시 반응형

데이터 변화 또는 리스크 발생 시, 즉시 판단하여 브리핑을 생성한다.

- 변화 감지 브리핑
- 리스크 감지 브리핑

---

#### 3. 상태 판단형

사용자의 활동 상태 또는 조건 충족 시점에 브리핑을 생성한다.

- 마무리 브리핑: 퇴근 시간 이후 또는 마지막 접속 1시간 경과 시 생성
- 장시간 미처리 기반 브리핑

---

## 3. 브리핑 우선순위 정책

### 3-1. 우선순위 기준

브리핑은 사용자 행동에 미치는 영향도를 기준으로 우선순위를 가진다.

| 우선순위 | 브리핑 유형 |
|----------|------------|
| 1 | 리스크 감지 브리핑 |
| 2 | 변화 감지 브리핑 |
| 3 | 일정 브리핑 |
| 4 | 오전 / 마무리 브리핑 |
| 5 | 사용자 설정 브리핑 |

---

### 3-2. 적용 방식

- 동일 시점에 여러 브리핑이 생성될 경우, 우선순위가 높은 브리핑이 먼저 노출된다.
- 푸시 알림은 우선순위 기준으로 순차 발송된다.

---

## 4. 데이터 처리 정책

### 4-1. 데이터 수집 범위

브리핑은 다음 데이터를 기반으로 생성된다.

#### Phase 1
- 플로우 업무 데이터
- 플로우 메시지 및 멘션
- 캘린더 일정

#### Phase 2
- 메일 데이터
- 외부 캘린더
- 외부 협업 도구

#### Phase 3
- 내부 + 외부 데이터 통합
- 데이터 간 관계 분석

---

### 4-2. 데이터 필터링 기준

- 마감 임박 업무 우선
- 일정과 연관된 업무 우선
- 사용자에게 직접 영향을 주는 정보 우선
- 불필요한 노이즈 데이터 제외

---

## 5. 브리핑 콘텐츠 중복 제어 정책

### 5-1. 기본 원칙

- 동일 항목은 반복 노출하지 않는다.
- 브리핑은 독립적이지만 콘텐츠는 공유된다.

---

### 5-2. 브리핑 유형별 기준

#### (1) 요약형 브리핑
- 대상: 오전, 마무리
- 기존 정보 재사용 가능
- 재정리 형태로 제공

---

#### (2) 이벤트형 브리핑
- 대상: 일정, 변화, 리스크
- 동일 항목 반복 금지
- 기존 정보 제외 또는 축약

---

### 5-3. 중복 판단 기준

- 동일 업무/일정/메시지 ID
- 동일 행동 요구
- 동일 의미의 정보

---

### 5-4. 예외 처리 (업데이트)

다음 경우는 중복이 아닌 업데이트로 처리

- 상태 변경 (진행 → 완료)
- 마감 임박
- 일정 변경

---

## 6. 리스크 감지 정책 (Phase 3)

### 6-1. 감지 대상

- 업무 지연
- 일정 충돌
- 마감 임박
- 업무 의존성 문제

---

### 6-2. 판단 기준

- 일정 대비 진행률
- 업무 간 의존 관계
- 시간 대비 작업량
- 사용자 참여 상태

---

### 6-3. 출력 원칙

리스크 브리핑은 다음 구조를 따른다.

- 문제 상황
- 영향 범위
- 추천 행동

---

## 7. 사용자 설정 브리핑 정책

### 7-1. 정의

사용자가 정의한 프롬프트를 특정 시간 또는 조건에서 실행하여 생성되는 브리핑

---

### 7-2. 실행 조건

- 특정 시간 (예: 매일 9시)
- 특정 이벤트 발생 시
- 반복 조건

---

### 7-3. 처리 원칙

- 시스템 브리핑과 독립적으로 실행
- 중복 콘텐츠 정책 동일 적용
- 필요 시 다른 브리핑과 병합 가능

---

## 8. 브리핑 생성 로직 요약

```
데이터 수집 → 필터링 → 우선순위 판단 → 중복 제거 → 브리핑 생성 → 전달
```

---

## 9. 기대 효과

- 정보 과부하 감소
- 사용자 의사결정 속도 향상
- 업무 누락 최소화
- AI 비서 신뢰도 향상
