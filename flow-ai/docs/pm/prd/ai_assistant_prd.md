## 1. 제품 개요

### 1-1. 제품 정의

AI 비서는  
플로우 및 연동된 업무 데이터를 기반으로 사용자의 업무 현황을 이해하고,
**상황에 맞는 브리핑을 선제적으로 제공하여 다음 행동을 유도하는 AI 비서**이다.

---

### 1-2. 문제 정의

현재 사용자는 다음과 같은 문제를 겪고 있다.

- 여러 프로젝트를 직접 확인해야 하는 **정보 탐색 비용**
- 마감 업무 및 멘션을 놓치는 **업무 누락 리스크**
- 일정 준비 및 업무 우선순위 판단에 드는 **인지 부담**
- 하루 업무를 정리하지 못하는 **맥락 단절**

---

### 1-3. 목표

#### 사용자 목표
- 출근 전 3분 이내 오늘 할 일 파악
- 중요한 업무를 놓치지 않음
- 하루 업무 흐름을 자연스럽게 관리

#### 비즈니스 목표
- 플로우 AI 사용 빈도(DAU) 증가
- AI 기능의 실사용 전환율 증가
- 업무 생산성 향상 기반 리텐션 강화

---

### 1-4. 성공 지표 (KPI)

| 지표 | 목표 | 측정 방법 |
|------|------|----------|
| 브리핑 확인율 | 70% 이상 | 브리핑 생성 대비 사용자 열람 비율 |
| 브리핑 기반 행동 전환율 | 30% 이상 | 브리핑 내 추천 행동 클릭/실행 비율 |
| AI 비서 DAU | 전체 DAU 대비 40% 이상 | AI 비서 화면 일일 방문자 수 |
| 브리핑 만족도 | 4.0/5.0 이상 | 브리핑 피드백 (좋아요/싫어요) 비율 |

---

## 2. 핵심 개념

### 2-1. 브리핑 중심 구조

> **AI 비서는 ‘브리핑을 생성하는 시스템’이다**

모든 기능은 브리핑 형태로 제공된다.

---

### 2-2. 브리핑 정의

브리핑은 특정 시점 또는 상황에서  
사용자에게 필요한 정보를 분석하여 전달하고  
다음 행동을 제안하는 AI 비서의 핵심 행위이다.

---

### 2-3. 브리핑 타입

| 타입 | 설명 | Phase |
|------|------|-------|
| 오전 브리핑 | 하루 시작 시, 일정·업무·변화 사항을 요약하여 업무 시작을 돕는 브리핑 | P1 |
| 일정 브리핑 | 일정 전에 관련 정보와 준비사항 정리를 돕는 브리핑 | P1 |
| 마무리 브리핑 | 하루 종료 시, 완료 업무와 미처리 업무를 정리하고 다음 행동을 제안하는 브리핑 | P1 |
| 사용자 설정 브리핑 | 사용자가 시간 또는 조건을 직접 설정하여 원하는 시점에 제공되는 브리핑 | P2 |
| 변화 감지 브리핑 | 업무·메시지·일정 등 주요 변화 발생 시 이를 감지하여 즉시 제공되는 브리핑 | P2 |
| 리스크 감지 브리핑 | 업무 흐름을 분석하여 병목, 지연, 우선순위 충돌 등의 리스크를 감지하고 선제적으로 대응이 필요한 업무를 제안하는 브리핑 | P3 |

---
### 2-4. 브리핑 콘텐츠 중복 제어 정책

브리핑은 타입별로 독립적으로 생성될 수 있으나,  
동일한 업무, 일정, 메시지, 변화 항목이 여러 브리핑에 반복 포함되지 않도록 제어한다.


#### 1. 기본 원칙

- 동일 시간대 또는 인접 시간대에 여러 브리핑이 생성될 수 있다.
- 각 브리핑은 독립적인 목적과 트리거를 가진다.
- 단, 동일한 핵심 항목은 불필요하게 반복 노출하지 않는다.

#### 2. 브리핑 유형별 적용 기준

브리핑의 성격에 따라 중복 제어 방식을 다르게 적용한다.

##### (1) 요약형 브리핑
- 대상: 오전 브리핑, 마무리 브리핑
- 특성: 전체 상황을 정리하는 브리핑

처리 원칙:
- 기존에 전달된 항목이라도 다시 포함할 수 있다.
- 단순 반복이 아닌 “재정리된 정보”로 제공해야 한다.


##### (2) 이벤트형 브리핑
- 대상: 일정 브리핑, 변화 감지 브리핑, 리스크 감지 브리핑
- 특성: 특정 이벤트 또는 트리거 기반 브리핑

처리 원칙:
- 동일 항목의 반복 노출을 금지한다.
- 이미 전달된 내용은 제외하거나 축약한다.


#### 3. 중복 판단 기준

다음 조건 중 하나 이상을 만족할 경우 동일 콘텐츠로 판단한다.

- 동일한 업무 ID, 일정 ID, 게시글 ID, 메시지 스레드 ID를 참조하는 경우
- 동일한 사용자 행동을 요구하는 경우
- 일정 시간 범위 내 의미적으로 동일한 정보인 경우

#### 4. 상태 변화 예외 처리

다음 조건에 해당하는 경우 중복이 아닌 “업데이트”로 간주한다.

- 업무 상태 변경 (예: 진행중 → 완료)
- 마감 임박 등 중요도 상승
- 일정 변경 또는 새로운 정보 추가

이 경우, 기존 항목을 재노출하되 변화된 내용 중심으로 표현한다.



#### 5. 적용 효과

- 브리핑 간 정보 중복 최소화
- 요약 브리핑의 정보 완결성 유지
- 사용자 인지 피로 감소

---
### 2-5. 브리핑 데이터 범위 (Phase 기반 확장)

브리핑에 활용되는 데이터 범위는 Phase에 따라 단계적으로 확장된다.  
초기에는 플로우 내부 데이터를 중심으로 시작하며, 이후 외부 데이터 및 다양한 서비스로 확장한다.


#### 1. Phase 1 — 내부 업무 데이터 기반

플로우 내 핵심 업무 데이터를 기반으로 브리핑 생성

- 플로우 업무 데이터
- 플로우 메시지 및 멘션
- 캘린더 일정 데이터

👉 특징
- 구조화된 데이터 중심
- 정확도 높음
- 기본 브리핑 기능 구현 가능


#### 2. Phase 2 — 확장 데이터 기반

사용자의 업무 맥락을 확장하기 위한 외부 데이터 연동

- 메일 데이터 (수신/발신 메일)
- 외부 캘린더 (Google, Outlook 등)
- 외부 협업 도구 데이터 (추후 확장)

👉 특징
- 업무 맥락 확장
- 브리핑 정확도 및 완성도 향상
- 사용자 설정 브리핑과 결합 가능


#### 3. Phase 3 — 통합 컨텍스트 기반

다양한 데이터 소스를 통합하여 분석 및 판단에 활용

- 내부 + 외부 데이터 통합
- 데이터 간 관계 분석 (업무-메일-일정 연결)
- 리스크 및 병목 분석

👉 특징
- 단순 데이터 활용을 넘어 “관계 기반 분석”
- 리스크 감지 및 우선순위 판단 가능
- 예측 기반 브리핑 구현

### 2-6. 브리핑 생성 기준 시점

모든 브리핑은 “현재 시점 (now)”을 기준으로 생성된다.  
브리핑에 포함되는 정보는 현재 시점에서 사용자에게 의미 있는 정보로 재구성된다.


#### 1. 기본 원칙

- 브리핑은 과거 이벤트의 단순 나열이 아니다.
- 현재 시점에서 사용자에게 필요한 정보만 선별하여 제공한다.
- 동일한 데이터라도 시점에 따라 중요도가 달라질 수 있다.


#### 2. 시점 기반 정보 재구성

브리핑은 다음과 같이 현재 시점을 기준으로 정보를 재해석한다.
- 과거 → 최근 변화로 요약
- 현재 → 즉시 행동이 필요한 항목 강조
- 미래 → 예정된 일정 및 준비사항 제안


#### 3. 적용 예시

예: 동일한 업무 데이터

- 오전 브리핑  
  → “오늘 마감 업무”로 표현

- 리스크 감지 브리핑  
  → “지금 처리해야 할 업무”로 표현

- 마무리 브리핑  
  → “오늘 미처리 업무”로 표현


#### 4. 핵심 원칙

브리핑은 데이터 중심이 아니라  
“현재 시점에서의 사용자 상황” 중심으로 생성된다.

## 3. 시스템 동작 구조

AI 비서는 브리핑을 생성하는 방식에 따라 단계적으로 고도화된다.  
각 Phase는 브리핑 생성의 트리거 방식과 분석 수준의 차이를 의미한다.

---

#### Phase 간 차이 요약

| 구분 | Phase 1 | Phase 2 | Phase 3 |
|------|--------|--------|--------|
| 트리거 | 시간 / 일정 | 변화 감지 / 사용자 정의 실행 | 분석 / 예측 |
| 역할 | 정보 요약 | 상황 대응 / 사용자 의도 실행 | 문제 예방 / 우선순위 개입 |
| 수준 | 정리 | 인식 + 실행 | 판단 + 개입 |
| AI 개입 | 낮음 | 중간 | 높음 |

---

#### 핵심 방향성

Phase가 발전할수록 AI 비서는 단순 정보 전달을 넘어  
사용자의 업무 흐름을 이해하고 개입하는 방향으로 진화한다.


---

### 3-1. Phase 1 — 정적 브리핑 (Scheduled Briefing)

정해진 시점 또는 명확한 이벤트를 기준으로 브리핑을 생성하는 단계

#### 특징
- 사전에 정의된 트리거 기반
- 구조화된 데이터 요약 중심
- 예측/판단 로직 최소

#### 포함 기능
- 오전 브리핑
- 일정 브리핑
- 마무리 브리핑

#### 동작 구조

```
데이터 수집 → 컨텍스트 생성 → 브리핑 생성 → 전달
```

---

### 3-2. Phase 2 — 상황 인식 브리핑 (Context-aware Briefing)

사용자의 업무 환경 변화 또는 사용자 정의 조건을 기반으로 브리핑을 생성하는 단계

---

#### 특징

- 이벤트 또는 사용자 정의 조건 기반 트리거
- 사용자 의도 반영
- 변화 감지 및 사용자 설정 실행

---

#### 포함 기능

##### (1) 변화 감지 브리핑
- 업무, 메시지, 일정 등의 변화를 시스템이 감지하여 생성

##### (2) 사용자 설정 브리핑
- 사용자가 정의한 프롬프트를 특정 시간 또는 조건에서 실행하여 생성

---

#### 동작 구조

```
(1) 변화 감지 브리핑 :
데이터 수집 → 변화 감지 → 브리핑 생성 → 전달
```

```
(2) 사용자 설정 브리핑 :
사용자 프롬프트 → 실행 조건 도달 → 브리핑 생성 → 전달
```

---

### 3-3. Phase 3 — 예측 기반 브리핑 (Predictive Briefing)

업무 흐름을 분석하여 리스크 및 병목을 사전에 감지하고 브리핑을 생성하는 단계

#### 특징
- 분석 및 추론 기반
- 리스크/우선순위 판단 포함
- 선제적 행동 유도

#### 포함 기능
- 리스크 감지 브리핑

#### 동작 구조

```
데이터 수집 → 상태 분석 → 리스크 판단 → 브리핑 생성 → 행동 제안 → 전달
```

## 4. 핵심 기능

### 4-1. 오전 브리핑

#### 개요
하루 시작 시, 오늘의 업무 상황을 요약하여 제공하는 브리핑

#### 제공 정보
- 오늘 일정
- 오늘 해야하는 업무
- 부재중 변화

#### 기대 효과
- 빠른 업무 시작
- 우선순위 명확화

---

### 4-2. 일정 브리핑

#### 개요
일정 시작 전, 관련 정보와 준비사항을 제공하는 브리핑

#### 제공 정보
- 일정 요약 (시간, 참석자)
- 관련 업무 및 자료
- 준비 필요 사항

#### 기대 효과
- 회의 준비 시간 단축
- 맥락 이해도 향상

---

### 4-3. 퇴근 브리핑

#### 개요
하루 종료 시, 업무를 정리하고 다음 행동을 제안하는 브리핑

#### 제공 정보
- 완료 업무 요약
- 미처리 업무
- 내일 일정

#### 기대 효과
- 업무 정리 습관 형성
- 다음날 준비 용이

---

## 5. 브리핑 스타일 (보조 기능)

### 5-1. 정의

브리핑 스타일은  
**동일한 정보를 사용자 성향에 맞게 다르게 전달하는 방식**이다.

---

### 5-2. 스타일 종류

| 스타일 | 설명 |
|------|------|
| 핵심형 | 핵심만 간결하게 요약 |
| 근거형 | 이유와 함께 분석 |
| 대화형 | 친근한 대화 방식 |
| 액션형 | 실행 중심 지시 |

---

### 5-3. 적용 방식

- 사용자 설정으로 선택 가능
- 모든 브리핑에 공통 적용
- 동일 데이터라도 스타일에 따라 출력 구조가 달라짐

---

## 6. 사용자 흐름

### 6-1. 온보딩

- 브리핑 스타일 선택
- 브리핑 항목 설정

---

### 6-2. 브리핑 수신

- 특정 시점/상황에서 브리핑 생성
- 푸시 알림 또는 AI 비서 홈 화면에서 확인

---

### 6-3. 행동 실행

- 추가 질의 입력
- 플로우/캘린더 등으로 연결

## 8. 비기능 요구사항

### 8-1. 성능

- 브리핑 생성 완료: 발송 시각 기준 30분 전 생성 시작
- 브리핑 응답 로딩: 사용자 화면 진입 후 3초 이내 표시

### 8-2. 보안 및 데이터

- 사용자 데이터는 본인의 브리핑 생성에만 사용
- 타 사용자의 업무/메시지 데이터 접근 불가
- 브리핑 생성에 사용된 원본 데이터는 브리핑 생성 후 별도 저장하지 않음

### 8-3. 안정성

- 데이터 수집 실패 시 해당 섹션을 제외하고 브리핑 생성 (부분 생성 허용)
- 브리핑 생성 실패 시 사용자에게 실패 안내 및 재시도 옵션 제공
