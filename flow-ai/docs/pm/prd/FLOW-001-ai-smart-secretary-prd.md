## 1. 개요

### 1-1. 제품 비전

> "Flow에 쌓인 업무 맥락을 누구보다 잘 이해하고, 사용자가 묻기 전에 먼저 챙겨주는 AI 업무 파트너"

AI 스마트 비서는 단순한 챗봇이나 알림 봇이 아닙니다. 협업 도구 Flow에 누적된 메시지, 일정, 업무, 파일 데이터를 바탕으로 사용자의 업무 상태를 파악하고, **능동적(proactive)**으로 정보를 제공하며 다음 행동을 유도하는 엔터프라이즈 AI 비서입니다.

### 1-2. 배경 및 문제 정의

**현황 문제**

| 문제 | 구체적 현상 |
|------|------------|
| 정보 과부하 | 사용자가 출근 후 수십 개의 채널·프로젝트를 직접 순회하며 업무를 파악해야 함 |
| 누락 리스크 | 마감 임박 업무, 미회신 메시지를 사용자가 능동적으로 체크하지 않으면 놓칠 수 있음 |
| 맥락 단절 | 장기 부재(휴가, 출장) 후 복귀 시 업무 현황 파악에 과도한 시간 소요 |
| 도구 분산 | 별도 캘린더·리마인더 앱을 병행 사용해야 하는 불편함 |

**기회**

Flow는 이미 메시지, 업무(Task), 일정, 채널 멤버십 등 풍부한 업무 데이터를 보유하고 있습니다. 이 데이터를 AI로 해석해 사용자에게 선제적으로 전달하면, 별도 도구 없이도 업무 파악과 실행이 Flow 안에서 완결될 수 있습니다.

### 1-3. 목표

**비즈니스 목표**
- Flow AI 구독 전환율 향상: AI 스마트 비서를 AI 플랜의 핵심 가치로 포지셔닝
- MAU 대비 DAU 비율(Stickiness) 개선: 매일 방문할 이유 제공
- 기업 고객 갱신율 향상: 업무 효율 개선을 통한 ROI 증명

**사용자 목표**
- 출근 후 3분 이내 오늘 해야 할 일 파악 가능
- 중요한 업무 누락 없이 처리
- Flow와 flow AI의 결합으로 업무 루틴 완결

## 2. 화면 구조 (IA)

### 2-1. 화면 목록

| 화면 ID | 화면명 | 진입 경로 | 설명 |
|---------|--------|----------|------|
| scr-000 | flow AI 메인 채팅 | 앱 진입 기본 화면 | 플로우 검색, 웹 검색, 이미지 생성, AI 챗봇 퀵 액션 |
| scr-onboard | AI 맞춤 설정 온보딩 | AI 비서 첫 진입 시 / 사이드바 프로필 | 캐릭터 선택(Step 1) + 대화 스타일 선택(Step 2) |
| scr-100 | AI 비서 홈 | 사이드바 > AI 비서, 푸시 알림 탭 | 선제형 메시지 피드 (프로필 카드 + 알림 카드 + 지난 대화) |

| scr-settings | 비서 설정 | scr-100 우상단 설정 아이콘 | 알림 설정 + AI 비서 설정 통합 단일 페이지 |

> - scr-scenario: 시나리오 선택 화면 (데모용 타임라인 — 오전 출근/일정 준비/퇴근 전/사이드바 진입)
> - scr-phone: 모바일 홈 화면 시뮬레이션 (푸시 알림 수신 데모용)

### 2-2. 주요 진입 플로우

```
[앱 진입]
  └── scr-000 (flow AI 메인)
        └── 사이드바 열기

                    └── (재진입) scr-100

[푸시 알림 수신]
  └── 알림 탭
        └── scr-110 (해당 채팅방 직접 진입)
```

## 3. 타겟 유저

### 3-1. 주요 사용자 페르소나

#### 페르소나 A: 바쁜 팀 리더 — "김과장" (35세, 마케팅팀 팀장)
- **상황**: 5~10명 팀 관리, 하루 평균 50건 이상 메시지 수신
- **고충**: 팀원 업무 진행 상황을 파악하느라 아침 시간을 소모, 자신의 업무는 뒷전
- **니즈**: 팀 전체 현황을 한눈에 파악하고 빠르게 의사결정하고 싶음
- **AI 비서 활용 포인트**: 아침 브리핑으로 팀 업무 요약, 미회신 메시지 우선순위 파악

#### 페르소나 B: 멀티태스킹 실무자 — "이대리" (28세, 개발팀 프론트엔드)
- **상황**: 3개 이상 프로젝트 동시 참여, 여러 채널에서 멘션 다수
- **고충**: 마감 관리 실패, 우선순위 혼란, 중요 메시지 놓침
- **니즈**: 지금 가장 급한 것을 알려주고, 잊지 않게 상기시켜줬으면 함
- **AI 비서 활용 포인트**: 지능형 리마인더, 마감 임박 알림

#### 페르소나 C: 복귀 직원 — "박주임" (32세, 인사팀)
- **상황**: 2주 육아휴직 후 복귀
- **고충**: 어디서부터 파악해야 할지 막막함, 중요한 변경사항을 놓칠 불안
- **니즈**: 내가 없는 동안 뭐가 바뀌었는지 빠르게 요약해줬으면 함
- **AI 비서 활용 포인트**: 오전 브리핑의 "부재중 변화" 섹션

### 3-2. 사용 시나리오

#### 시나리오 1: 오전 출근 (AM 8:50)
```
1. 이대리가 AM 8:50 Push 알림 수신 (AI 비서 아침 브리핑)
2. 알림 탭 → scr-110(브리핑 채팅방) 직접 진입
3. 브리핑: 오늘 일정 3건, 마감 업무 2건, 밤사이 변경 10건 요약
4. 이대리가 마감 업무 항목선택 > 플로우앱 해당 게시글 이동
5. 채팅 세션에서 바로 이어서 자유 대화 가능
```

#### 시나리오 2: 일정 준비 알림 (PM 1:30)
```
1. 김과장이 PM 1:30 Push 알림 수신 (스프린트 리뷰 30분 전)
2. 알림 탭 → scr-110(일정 준비 채팅방) 진입
3. AI가 회의 정보, 준비 사항, 관련 업무 요약 제공
4. 채팅 세션에서 바로 이어서 자유 대화 가능
```

#### 시나리오 3: 퇴근 마무리 리포트 (PM 7:30 이후 행동 기반)
```
1. 이대리가 PM 6:00 이후 Flow 마지막 접속 후 1시간 경과
2. AI 비서가 퇴근 마무리 리포트 Push 알림 발송
3. 알림 탭 → scr-110(마무리 채팅방) 진입
4. 오늘 처리 업무 요약, 미처리 업무, 내일 할 일 정리 확인
5. 채팅 세션에서 바로 이어서 자유 대화 가능
```

#### 시나리오 4: 사이드바 직접 진입
```
1. 박주임이 Flow AI 앱 내에서 사이드바 열기
2. 사이드바 > AI 비서 탭
3. (첫 진입) scr-onboard → 캐릭터/대화 스타일 선택 → scr-100
4. scr-100에서 미확인 알림 카드 목록 확인
5. 카드 탭 → scr-110 해당 채팅방 진입
```

## 5. 기능 요구사항

### 5-1. 온보딩

#### 개요
신규 사용자가 AI 스마트 비서를 처음 활성화할 때 진행하는 2단계 설정 흐름입니다. 목표 소요시간은 30초 이내입니다. 온보딩 완료 후에도 설정 화면(scr-settings)에서 캐릭터·대화 스타일 변경이 가능합니다.

#### 온보딩 진입 조건
- AI 비서를 처음 진입하는 경우 (localStorage: `flowai_customized` 미존재)
- 온보딩 중 X 버튼으로 닫으면 시나리오 선택 화면(또는 이전 화면)으로 복귀
- 온보딩 진입 전 pendingScenario를 저장하여, 완료 후 해당 시나리오 화면으로 자동 이동

#### 유저 스토리
- 사용자는 AI 스마트 비서를 처음 시작할 때 나만의 AI 캐릭터를 선택할 수 있다.
- 사용자는 AI 캐릭터의 대화 스타일을 선택할 수 있다.
- 사용자는 온보딩 중 이전 단계로 돌아갈 수 있다.
- 사용자는 온보딩을 건너뛸 수 있으며, 이 경우 기본값(플로키 + 기본적인 스타일)이 자동 적용된다.

#### Step 1: AI 캐릭터 선택

사전 정의된 3종의 캐릭터 중 1개를 선택합니다.

| 캐릭터 ID | 이름 | 이모지 | 설명 | 그라데이션 | 비고 |
|----------|------|--------|------|-----------|------|
| floki | 플로키 | 🐙 | 호기심 많은 바다 탐험가 | 핑크 (#FFD1E8 → #FFA3C7) | "인기" 태그 표시, 기본값 |
| penguin | 펭귀니 | 🐧 | 남극에서 온 업무 도우미 | 블루 (#D1E8FF → #8EC5FC) | — |
| turtle | 거북이 | 🐢 | 믿음직한 숲속 길잡이 | 그린 (#D1FFE8 → #8EFCB3) | — |

- 캐릭터 카드 선택 시 테두리 강조 + 배경색 변경 (selected 상태)
- 1개 이상 선택 시 "다음" 버튼 활성화
- 하단 "나중에 설정할게요" 버튼: 스킵 처리 (기본값 적용 후 onboarding 완료 처리)

#### Step 2: 대화 스타일 선택

7종의 대화 스타일 중 1개를 선택합니다.

| 스타일 ID | 라벨 | 설명 | 기본값 |
|----------|------|------|--------|
| default | 기본적인 | 기본 스타일과 말투 | 기본값 (✓ 표시) |
| serious | 진중한 | 정제되어 있고 정확한 답변을 해줘요 | — |
| friendly | 친근한 | 따뜻하고 수다스러운 답변을 해줘요 | — |
| candid | 솔직한 | 직설적이면서도 격려하는 답변을 해줘요 | — |
| creative | 독특한 | 유쾌하고 상상력이 풍부한 답변을 해줘요 | — |
| concise | 간결한 | 간결하고 꾸밈없는 답변을 해줘요 | — |
| snarky | 까칠한 | 비꼬면서 비판적인 답변을 해줘요 | — |

- 화면 상단: Step 1에서 선택한 캐릭터 아바타 + "{캐릭터 이름}의 대화 스타일을 골라주세요" 표시
- 리스트 형태로 표시 (라벨 bold + 설명 sub-text), 선택 시 ✓ 체크 표시
- 기본값이 사전 선택된 상태로 시작, 다른 항목 선택 시 변경
- "시작하기" 버튼 항상 활성화 (기본값이 사전 선택되어 있으므로)
- 하단 "캐릭터 다시 선택" 버튼: Step 1으로 이동

#### 온보딩 완료 처리
- `flowai_character`, `flowai_style`, `flowai_customized: true` 저장 (localStorage → 서버 동기화)
- 완료 후 pendingScenario가 있으면 해당 화면으로 이동, 없으면 scr-100(AI 비서 홈)으로 이동
- 온보딩 완료 상태 서버 저장 → 재접속 시 온보딩 스킵

#### 스킵(나중에 설정) 처리
- 기본값 자동 적용: 캐릭터 = 플로키(floki), 대화 스타일 = 기본적인(default)
- 완료 처리와 동일하게 저장 후 화면 이동
- 설정 화면에서 언제든 변경 가능

#### 예외 케이스

| 상황 | 처리 방법 |
|------|----------|
| X 버튼으로 온보딩 닫기 | 저장하지 않고 이전 화면(시나리오 선택 또는 scr-000)으로 복귀 |
| pendingScenario 있을 때 닫기 | pendingScenario 초기화 후 scr-scenario로 복귀 |
| 온보딩 중 앱 강제 종료 | 미완료 상태 유지 → 재접속 시 온보딩 재시작 |
| 네트워크 오류로 서버 저장 실패 | 로컬 저장 우선 처리, 백그라운드 재시도 |

### 5-2. AI 비서 홈 (scr-100)

#### 개요
AI 비서의 메인 화면입니다. 사용자에게 선제적으로 알림을 전달하는 피드 형태로 구성되며, 미확인 알림 카드와 지난 대화 섹션으로 나뉩니다.

#### 화면 구성

**헤더**
- 좌: 햄버거(사이드바 열기) + "AI 비서" 텍스트 + 시나리오 시계 (시나리오 진입 시 표시)
- 우: 설정 아이콘 → scr-settings 이동

**프로필 카드 (상단 고정)**
- 캐릭터 아바타 (선택한 캐릭터 이모지 + 그라데이션 배경)
- 인사 문구: 시간대별 감성 인사 (랜덤 1건 표시), 2줄 구성 (인사 + 맥락 정보)

**시간대별 인사 문구 풀**

| 시간대 | 인사 문구 예시 |
|--------|-------------|
| 오전 | "{이름}님, 오늘 서울 날씨는 맑고 18°C예요. \n 가벼운 겉옷 하나 챙기세요!" |
| 오전 | "{이름}님, 좋은 아침이에요! \n 오늘 일정이 꽤 있으니 커피 한 잔 어때요?" |
| 오전 | "{이름}님, 오늘 하루도 힘내요! \n 오전에 중요한 미팅이 있어요." |
| 오후 | "{이름}님, 점심은 드셨나요? \n 오후 일정도 잘 마무리해봐요!" |
| 오후 | "{이름}님, 오후에도 파이팅! \n 마감 업무가 있으니 체크해보세요." |
| 오후 | "{이름}님, 벌써 오후네요. \n 남은 업무 함께 정리해볼까요?" |
| 저녁 | "{이름}님, 오늘 하루 수고 많으셨어요! \n 내일 일정도 미리 확인해볼까요?" |
| 저녁 | "{이름}님, 퇴근 전 마무리할 것들 정리해드릴게요." |
| 저녁 | "{이름}님, 굿 이브닝! \n 오늘 처리한 업무를 정리해뒀어요." |

**새 알림 카드 섹션**
- 미확인 알림 순서: 시간 역순 정렬
- 각 카드 좌상단: 미확인 파란 dot 표시
- 탭 시 scr-110(채팅방 상세) 진입, 읽음 처리 (파란 dot 제거)
- 알림 종류: 오전 브리핑, 일정 준비 알림(개별), 퇴근 마무리 리포트

**지난 대화 섹션**
- 확인 완료된 항목 및 과거 히스토리
- 섹션 타이틀: "지난 대화"
- 카드 형태 동일, 읽음 상태 (dot 없음)

#### 유저 스토리
- 사용자는 한 화면에서 오늘 받은 모든 AI 비서 알림을 확인할 수 있다.
- 사용자는 알림 카드를 탭하여 해당 채팅방으로 진입할 수 있다.
- 사용자는 프로필 카드에서 자신이 선택한 AI 캐릭터를 확인할 수 있다.

### 5-3. 오전 브리핑

#### 개요
매일 아침 AI 비서가 사용자에게 자동으로 보내는 프로액티브 메시지입니다. 브리핑 발송 시 새로운 채팅 세션이 생성되며, 사용자는 Push 알림을 탭하거나 scr-100의 알림 카드에서 해당 세션으로 진입할 수 있습니다. Flow의 업무·캘린더 데이터를 LLM이 해석하여 자연어 요약으로 생성하며, 3개 섹션 + 추천 순서로 구성됩니다. 각 섹션 응답 하단에는 AI 퀵 액션(제안 칩)이 제공되어 사용자가 후속 작업을 이어갈 수 있습니다.

> 섹션별 데이터 수집 범위, 필터 조건, 중복 판단 규칙은 [브리핑 데이터 수집·분류 스펙](../specs/FLOW-001-briefing-data-spec.md) 참조

#### 데이터 수집 타이밍 정책
- **수집 시작**: 알림 발송 시각 30분 전 (예: 08:00 알림 설정 → 07:30부터 수집 시작)
- **수집 시각 동적 계산**: 설정된 브리핑 시각 기준으로 자동 계산 (`updateBriefingCollectHint` 처리)
- **설정 화면 힌트**: "📡 {수집 시작 시각}부터 일정·업무·변경사항을 수집해요" 표시
- **캐시 정책**: 동일 사용자 동일 날짜 브리핑은 캐시 우선 제공 (cache key = 사용자 ID + 날짜 + 기능 타입)
- **캐시 무효화**: 브리핑 발송 후 30분 이내 주요 데이터 변경(신규 긴급 업무 배정 등) 시 재생성 알림

#### Flow 데이터 소스

**업무 데이터**
| 필드 | 값 |
|------|----|
| 담당자 | 사용자 계정 |
| 시작일 / 마감일 | 날짜 |
| 상태 | 대기 / 진행 / 완료 / 보류 |
| 우선순위 | 낮음 / 보통 / 높음 / 긴급 |
| 업무 유형 | 상위업무 / 하위업무 |
| 상위업무명 | 텍스트 (하위업무인 경우에만 존재) |

**캘린더 데이터**
| 필드 | 값 |
|------|----|
| 일시 | 날짜 + 시간 |
| 참석자 | 사용자 계정 목록 |
| 장소 | 텍스트 (선택) |
| 내용 | 텍스트 (선택) |
| 외부 미팅 여부 | boolean (외부 참석자 포함 시 true) |

**댓글 / 멘션 데이터**
| 필드 | 값 |
|------|----|
| 대상 게시물 유형 | 글 / 업무 / 일정 |
| 대상 게시물 제목 | 텍스트 |
| 작성자 | 사용자 계정 |
| 작성 시각 | 날짜 + 시간 |
| 댓글 유형 | 댓글 / 대댓글 구분 |
| 멘션 여부 | 나를 멘션했는지 여부 (boolean) |
| 내 게시물 여부 | 댓글이 달린 게시물의 작성자가 나인지 여부 (boolean) |
| 내 댓글 대댓글 여부 | 내가 작성한 댓글에 달린 대댓글인지 여부 (boolean) |
| 댓글 내용 | 텍스트 원문 (LLM 요약용으로 전달) |

**LLM에 전달하는 댓글 범위**
- 나를 멘션한 댓글·대댓글 (게시물 유형 무관)
- 내가 작성한 게시물(글·업무·일정)에 달린 댓글
- 내가 작성한 댓글에 달린 대댓글

> LLM 응답에는 댓글 원문을 그대로 노출하지 않고 요약된 형태로만 제공. Prompt Guard 마스킹 정책 적용.

#### 브리핑 발송 정책
- **기본 발송 시간**: 평일 오전 08:00 (사용자 기기 타임존 기준)
- **설정 가능 범위**: 07:00 ~ 09:30 (30분 단위, 설정 화면에서 변경)
- **세션 생성**: 발송 시각에 새 채팅 세션 자동 생성, AI 브리핑 메시지가 첫 메시지로 기록됨
- **Push 알림**: 세션 생성과 동시에 발송 → 탭 시 scr-110 해당 세션으로 이동
- **발송 조건**: 전날 또는 당일 Flow 접속 이력이 있는 사용자에게만 발송
- **미발송 조건**: 3개 섹션 모두 표시할 데이터가 없는 경우 세션 미생성, 알림 없음

#### 유저 스토리
- 사용자는 매일 아침 설정된 시간에 브리핑 채팅 세션을 받을 수 있다.
- 사용자는 브리핑 세션에서 AI 퀵 액션(제안 칩)을 탭하여 후속 작업을 이어갈 수 있다.
- 사용자는 브리핑 세션에서 바로 이어서 자유롭게 대화할 수 있다.
- 사용자는 브리핑이 필요 없을 때 설정에서 비활성화할 수 있다.

#### 3개 섹션 상세 정의

**AI 퀵 액션 (제안 칩) 동작 원칙**

각 섹션 AI 응답 하단에 퀵 액션 칩이 제공됩니다. 칩을 탭하면 해당 항목과 관련된 후속 작업을 AI가 수행합니다.

```
AI 응답 (브리핑 섹션)
  └── 퀵 액션 영역: "⇢ 어떻게 도와드릴까요?"
        ├── [제안서 핵심 요약 정리해줘]
        ├── [기술 검토 자료 만들어줘]
        └── [리마인드 메시지 보내기]
              └── 탭 시: 선택한 칩 강조, 나머지 비활성화 → AI 후속 응답
```

- 퀵 액션 칩은 맥락에 따라 동적으로 생성 (일정/업무/변화 항목별 다른 제안)
- 칩 선택 시 해당 칩만 강조 표시, 나머지는 비활성화 처리
- AI 후속 응답 시 Flow 데이터 검색 → 스텝 프로그레스 애니메이션 → 자연어 요약

---

**섹션 1: 오늘의 일정**
- 데이터 소스: Flow 캘린더 — 당일 일시 기준 이벤트
- LLM 역할: 일시·장소·참석자를 자연어 한 줄 요약으로 변환
  - 예) "오후 3시, 본사 3층 회의실에서 김팀장 외 3명과 스프린트 리뷰 예정"
  - 외부 미팅: 외부 참석자 포함 시 회사명 표시
- 표시 건수: 최대 4건 (시작 시간 오름차순), 초과 시 "+N개 더" 표시
- 퀵 액션: AI 응답 하단에 맥락별 제안 칩 (예: "제안서 핵심 요약", "참석자에게 리마인드 메시지")
- 빈 상태: "오늘 예정된 일정이 없어요"

**섹션 2: 마감 업무**
- 데이터 소스: Flow 업무 — 내가 담당자이고 마감일이 당일 또는 D+1인 업무 (상태: 대기·진행)
  - 상위업무·하위업무 모두 포함
- LLM 역할: 업무명·마감일·우선순위를 조합하여 우선순위 순으로 자연어 요약
  - 예) "오늘까지 '[긴급] Q2 마케팅 보고서 초안'을 완료해야 해요"
- 하위업무 경로 표시: 상위 프로젝트 > 상위업무 > 하위업무명 계층 표시
  - 예) FlowAI QA > 스마트비서 기획 > PRD 작성
- 우선순위 배지: 긴급(빨강 🚨), 높음(주황 🟠), 보통(초록 🟢), 낮음(파랑 ➖)
- 표시 건수: 최대 5건 (긴급 → 높음 → 보통 → 낮음 순 정렬)
- 퀵 액션: AI 응답 하단에 맥락별 제안 칩 (예: "보고서 초안 작성", "업무 현황 정리")
- 빈 상태: "오늘과 내일 마감인 업무가 없어요"

**섹션 3: 부재중 변화**
- 데이터 소스: 전날 마지막 접속 이후 ~ 브리핑 생성 시각까지의 변경사항
  - 내가 담당자로 새로 배정된 업무
  - 내 업무의 상태·우선순위 변경
  - 나를 멘션한 댓글·대댓글 (글·업무·일정 게시물 모두 포함)
  - 내가 작성한 게시물에 달린 댓글
  - 내가 작성한 댓글에 달린 대댓글
- 변경 유형 아이콘: 신규 업무 배정 / 상태 변경 / 멘션 구분 표시
- LLM 역할: 변경 건수 및 댓글 내용을 종합하여 "놓친 것" 중심으로 자연어 요약
  - 예) "자는 동안 꽤 많은 일이 있었네요. 김팀장님이 새 업무를 맡겨두셨고, 박개발님 쪽 API 연동이 리뷰 단계로 넘어갔어요."
- 퀵 액션: AI 응답 하단에 맥락별 제안 칩 (예: "변경 내용 요약", "댓글 확인")
- 빈 상태: "놓친 변화가 없어요"

#### 세션 내 이어서 대화

브리핑 세션은 AI 첫 메시지 이후 사용자가 바로 이어서 대화할 수 있는 구조입니다.

```
[브리핑 채팅 세션]
  AI 첫 메시지: 3개 섹션 브리핑 + 추천 순서
  └── 사용자 입력 가능
        예) "마감 업무 중 가장 급한 거 먼저 알려줘"
            "어젯밤 팀에서 무슨 결정이 있었어?"
      └── AI 응답 (브리핑 컨텍스트 유지, 마크다운 렌더링)
            └── AI 퀵 액션 칩 ("⇢ 어떻게 더 도와줄까요?")
```

- 브리핑 데이터는 해당 세션의 컨텍스트로 LLM에 자동 전달됨 (사용자에게 비표시)
- 대화 기록은 해당 세션의 채팅 히스토리에 누적 저장
- AI 응답에 피드백 버튼 제공: 좋아요 / 싫어요 / 복사 / 공유

#### 예외 케이스
| 상황 | 처리 방법 |
|------|----------|
| LLM 생성 실패 | 오류 메시지 표시 + [다시 생성] 버튼. 재시도 후에도 실패 시 "잠시 후 다시 시도해주세요" 안내 |
| 3개 섹션 모두 데이터 없음 | 브리핑 미발송, 알림 없음 |
| 주말·공휴일 | 브리핑 미발송 (기본값), 설정에서 발송 허용 가능 (V2) |
| 사용자가 브리핑 수신 거부 설정 | 발송 중단, 설정 화면에서 재활성화 가능 |
| 타임존 변경 (해외 출장 등) | 기기 타임존 변경 감지 시 다음날부터 새 타임존 적용 |
| Prompt Guard 감지 | 해당 항목 마스킹 처리 후 요약, 뱃지 표시 |

### 5-4. 일정 리마인더

#### 개요
일정 시작 전 자동으로 알림을 보내는 리마인더입니다. 단순히 일정 정보를 알려주는 것이 아니라, 일정에 등록된 제목·참여자·본문을 참고하여 **플로우·위키·웹검색**을 수행하고 일정 관련 참고사항을 제공합니다. 리마인더 발송 시 새로운 채팅 세션이 생성되며, Push 알림 탭 시 scr-110 해당 세션으로 이동합니다.

#### 유저 스토리
- 사용자는 일정 시작 전 준비 알림을 받을 수 있다.
- 사용자는 일정 관련 참고사항(관련 자료, 이전 회의록 요약 등)을 AI가 검색·정리해서 받을 수 있다.
- 사용자는 일정 준비 알림 사전 알림 시간을 설정할 수 있다 (10분/15분/30분/1시간 전).
- 사용자는 리마인더를 스누즈하거나 해제할 수 있다.

#### 트리거 조건

| 조건 | 발송 타이밍 | 기본값 |
|------|------------|--------|
| 일정 시작 전 | 사전 알림 시간 기준 | 30분 전 |

- 대상: 내가 참석자로 등록된 캘린더 일정
- 사전 알림 시간 선택: 10분 전 / 15분 전 / 30분 전 (기본) / 1시간 전
- 알림 내용: 일정명, 시작 시각, 장소(있는 경우), 참석자 수
- 외부 미팅: 회사명 표시

#### AI 참고사항 제공 로직

일정 정보를 바탕으로 AI가 자동 검색하여 참고사항을 생성합니다.

```
[일정 정보 입력]
  ├── 제목, 참여자, 본문
  └── AI가 자동 검색 수행
        ├── 플로우 검색 (관련 업무·게시글·이전 회의록)
        ├── 위키 검색 (관련 문서·자료)
        └── 웹 검색 (외부 미팅 시 고객사 정보 등)
              └── 검색 결과 종합 → 참고사항 자연어 요약
```

- 내부 일정: 플로우·위키에서 관련 자료 검색 후 회의 준비 사항 정리
- 외부 미팅: 고객사 정보, 기존 제안서, 관련 자료 분석 제공
- 퀵 액션: 맥락별 제안 칩 (예: "제안서 핵심 요약", "참석자에게 리마인드 메시지")

#### 알림 피로 방지
- **방해금지 모드**: 설정 화면에서 활성화 시 모든 AI 비서 알림 일시 중지
- **동일 일정 중복 알림 방지**: 동일 일정에 대한 리마인더는 1회만 발송
- 방해금지 모드 ON 시 배지 업데이트 없음, scr-100에서 카드만 쌓임

#### 스누즈 규칙

| 스누즈 옵션 | 재발송 시점 |
|------------|------------|
| 10분 후 | 현재 시각 + 10분 |
| 30분 후 | 현재 시각 + 30분 |
| 해제 (이 일정 알리지 않기) | 해당 일정 리마인더 중단 |

#### 예외 케이스
| 상황 | 처리 방법 |
|------|----------|
| 일정이 취소된 경우 발송 직전 확인 | 발송 취소 |
| 방해금지 모드 ON | 알림 발송 중단, scr-100 카드만 생성 |
| 리마인더 발송 중 네트워크 오류 | 지수 백오프로 최대 3회 재시도 후 실패 처리 |

> **V2 확장 예정**: 할당된 업무·멘션 기반 리마인더, 커스텀 리마인더는 V2에서 추가

### 5-5. 퇴근 마무리 리포트

#### 개요
고정 발송 시간이 아닌 **행동 기반 트리거**로 작동하는 하루 마무리 리포트입니다. 사용자가 퇴근했다고 판단되는 시점에 자동으로 생성되어 Push 알림으로 전달됩니다.

#### 트리거 조건
- **기준 시각**: 18:00 이후
- **조건**: 사용자의 마지막 Flow 접속으로부터 1시간 경과
- **조합**: 18:00 이후 + 마지막 접속 후 1시간 경과 두 조건 모두 충족 시 발송

> 예시: 사용자가 18:30에 마지막으로 Flow를 사용했다면 19:30에 퇴근 마무리 리포트 발송

#### 데이터 수집 타이밍
- **수집 시점**: 퇴근 트리거 발생 시점 (마지막 접속 시각 기준)
- **수집 범위**: 퇴근 시점의 업무 현황 + 내일 일정
- **설정 화면 힌트**: "📡 퇴근 시점의 업무 현황·내일 일정을 수집해요"

#### 리포트 콘텐츠
- 오늘 처리 완료한 업무 건수 및 요약
- 미처리 업무 목록 (마감 임박 순)
- 내일 예정 일정 미리 보기
- 팀 채널 오늘의 주요 변화 (필요 시)

#### 유저 스토리
- 사용자는 퇴근 전 하루 업무를 정리하는 리포트를 자동으로 받을 수 있다.
- 사용자는 설정에서 퇴근 마무리 리포트를 비활성화할 수 있다.

#### 예외 케이스
| 상황 | 처리 방법 |
|------|----------|
| 18:00 이후 재접속 (야간 업무) | 마지막 접속 시각 갱신, 1시간 타이머 재시작 |
| 당일 접속 이력 없음 | 리포트 미발송 |
| 방해금지 모드 ON | 발송 중단, 다음날 발송 없음 (당일 건은 scr-100에만 카드 생성) |

### 5-6. 채팅방 상세 (scr-110)

#### 개요
브리핑, 리마인더, 퇴근 마무리 리포트 각각 독립적인 채팅 세션으로 관리됩니다.

#### 화면 구성

**헤더**
- 좌: 뒤로 가기 버튼 → scr-100 복귀
- 캐릭터 아바타 (원형, 선택한 캐릭터 이모지)
- 채팅방 제목 (예: "오전 브리핑", "스프린트 리뷰 준비", "마무리 리포트")
- 부제: "AI 비서"

**채팅 영역**
- AI 메시지: 전체 폭 레이아웃, 마크다운 렌더링 (marked.js)
  - h3, h4, ul, ol, blockquote, strong, em, code, hr 지원
- 사용자 메시지: 우측 말풍선 (배경: brand-500)
- Flow 검색/조회 블록: 스텝 프로그레스 애니메이션으로 데이터 수집 과정 시각화
- AI 피드백 버튼: 좋아요 / 싫어요 / 복사 / 공유 (AI 응답 하단)
- AI 제안 칩: "어떻게 더 도와줄까요?" 섹션 + 문맥 기반 추천 칩
- Prompt Guard 뱃지: LLM 요청 시 상단 표시

**입력 바**
- 텍스트 입력 필드: "무엇이든 물어보세요."
- 좌측 액션: 파일 첨부(+), AI 브레인, 설정
- 우측: AI 모델 선택 드롭다운 + 전송 버튼
- 전송 버튼: 입력 없을 때 비활성화 (opacity 50%, disabled)
- 하단 안내: "플로우 AI의 대화는 LLM에 학습·기록되지 않으며, 프롬프트 가드로 안전하게 처리됩니다."

### 5-7. 비서 설정 (scr-settings)

#### 개요
알림 설정과 AI 비서 설정을 하나의 페이지에 통합한 단일 설정 화면입니다. 스크롤 형태로 모든 설정을 한 화면에서 처리합니다.

#### 화면 구성

**헤더**
- 좌: 뒤로 가기 버튼 → scr-100 복귀
- 타이틀: "비서 설정"

**섹션 1: 알림 설정**

타이틀 폰트: 14px bold, 설명 폰트: 12px

| 항목 | 기본값 | 설정 가능 내용 |
|------|--------|--------------|
| 오전 브리핑 | ON, 08:00 | 토글 ON/OFF + 시간 선택 (07:00~09:30, 30분 단위) |
| 일정 준비 알림 | ON, 30분 전 | 토글 ON/OFF + 사전 알림 시간 선택 (10분/15분/30분/1시간) |
| 퇴근 마무리 리포트 | ON | 토글 ON/OFF (트리거 조건은 고정 표시) |
| 방해금지 모드 | OFF | 토글 ON/OFF |

**오전 브리핑 설정 상세**
- 토글 OFF 시: 시간 선택 섹션 접기 (숨김)
- 시간 선택: select 박스 (07:00 / 07:30 / 08:00 기본 / 08:30 / 09:00 / 09:30)
- 데이터 수집 힌트: "📡 {시각-30분}부터 일정·업무·변경사항을 수집해요" (동적 계산)

**일정 준비 알림 설정 상세**
- 토글 OFF 시: 사전 알림 시간 선택 섹션 접기
- 사전 알림 시간: select 박스 (10분 전 / 15분 전 / 30분 전 기본 / 1시간 전)

**퇴근 마무리 리포트 설정 상세**
- 트리거 조건 고정 표시: "18:00 이후 마지막 접속 1시간 뒤 알림"
- 데이터 수집 힌트: "📡 퇴근 시점의 업무 현황·내일 일정을 수집해요"

**방해금지 모드**
- ON 시 모든 AI 비서 알림(오전 브리핑, 일정 준비, 퇴근 리포트, 리마인더) 일시 중지

**섹션 2: AI 비서 설정**

타이틀 폰트: 14px bold

**AI 캐릭터 서브섹션**
- 헤더: "AI 캐릭터" (12px, uppercase, tracking-wider)
- 현재 캐릭터 표시: 아바타(64px, 캐릭터 그라데이션 배경) + 이름(14px bold) + 설명(12px)
- 캐릭터 변경 칩: 3종 캐릭터 이모지 칩 나열 (선택된 칩 강조)
- 선택 시 즉시 반영: 아바타, 이름, 설명 업데이트

**대화 스타일 서브섹션**
- 헤더: "대화 스타일" (12px, uppercase, tracking-wider)
- 7종 스타일 리스트 표시 (라벨 bold + 설명 sub-text + 선택 시 ✓ 체크)
- 선택 시 즉시 반영 (selected 상태 토글)
- 변경은 다음 AI 응답부터 즉시 적용

#### 예외 케이스
| 상황 | 처리 방법 |
|------|----------|
| 설정 변경 저장 실패 | 로컬 상태 유지, 백그라운드 재시도, Toast 알림 표시 |
| 토글 OFF 상태에서 시간 변경 시도 | 비활성화 처리 (UI 회색 처리) |

### 5-8. AI 캐릭터 및 대화 스타일 (공통 데이터)

#### AI 캐릭터 정의

| ID | 이름 | 이모지 | 설명 | 그라데이션 (배경) | dot 배경색 |
|----|------|--------|------|-----------------|-----------|
| floki | 플로키 | 🐙 | 호기심 많은 바다 탐험가 | #FFD1E8 → #FFA3C7 | #FFE4F0 |
| penguin | 펭귀니 | 🐧 | 남극에서 온 업무 도우미 | #D1E8FF → #8EC5FC | #E0EEFF |
| turtle | 거북이 | 🐢 | 믿음직한 숲속 길잡이 | #D1FFE8 → #8EFCB3 | #E0FFE8 |

- 캐릭터 설명은 정체성(identity) 기반 — 대화 스타일과 분리
- 캐릭터는 UI 표시용 (아바타, 이름, 인사 문구)으로 사용
- 기본 캐릭터: 플로키 (인기 태그 표시)

#### 대화 스타일 정의

| ID | 라벨 | 프롬프트 파라미터 | 설명 | 응답 특성 |
|----|------|-----------------|------|---------|
| default | 기본적인 | `style: default` | 기본 스타일과 말투 | 자연스럽고 균형 잡힌 답변, 상황에 맞게 톤 자동 조절 |
| serious | 진중한 | `style: serious` | 정제되어 있고 정확함 | 격식 있는 표현, 데이터 기반 정확한 답변, 불필요한 감정 표현 최소화 |
| friendly | 친근한 | `style: friendly` | 따뜻하고 수다스러움 | 대화체 중심, 공감 표현 적극 사용, 부드러운 톤 |
| candid | 솔직한 | `style: candid` | 직설적이면서도 격려함 | 핵심을 직접 전달, 문제점 명확히 지적하되 해결책과 격려 병행 |
| creative | 독특한 | `style: creative` | 유쾌하고 상상력이 풍부함 | 비유·예시 적극 활용, 창의적 관점 제시, 재치 있는 표현 |
| concise | 간결한 | `style: concise` | 간결하고 꾸밈없음 | 최소한의 문장, 불릿 포인트 선호, 부가 설명 생략 |
| snarky | 까칠한 | `style: snarky` | 비꼬면서 비판적임 | 위트 있는 독설, 날카로운 지적, 유머와 비판의 균형 |

- 기본 스타일: 기본적인 (default)
- 스타일 변경은 다음 AI 응답부터 즉시 적용
- 향후 Flow AI 전체 공통 설정으로 통합 예정 (현재는 AI 비서 전용)

## 7. 비기능 요구사항

### 7-1. 성능
- 브리핑 LLM 생성 응답 시간: P95 기준 10초 이내
- 채팅 응답 스트리밍: 첫 토큰 3초 이내 도달
- Flow 검색/조회 블록: 스텝별 애니메이션으로 수집 과정 시각화 (사용자 체감 대기 시간 감소)

### 7-2. 보안 및 개인정보

**Prompt Guard**
- 자동 마스킹 대상: 이름, 이메일, 전화번호, 주민등록번호, 신용카드 번호
- 금지어 차단: 시스템 프롬프트 탈취 시도, 역할극 공격, 탈옥 패턴 차단
- 처리 방식: LLM 전달 전 마스킹 → 응답 후 원복
- 채팅 UI: Prompt Guard 활성 상태 뱃지 표시

**데이터 격리**
- 사용자 데이터는 소속 조직 범위 내에서만 접근 가능
- 타 사용자 데이터 절대 열람 불가 (쿼리 레이어에서 사용자 ID 필터 강제 적용)
- AI 생성 브리핑에 타인의 개인 데이터가 포함되지 않도록 필터링

**로그 및 감사**
- 모든 LLM 요청·응답 로그 90일 보관
- 개인정보 마스킹 후 로그 저장
- GDPR / 개인정보보호법 준수 (사용자 데이터 삭제 요청 처리 7일 이내)

### 7-3. 접근성
- 모든 대화형 요소에 aria-label 제공
- 최소 터치 타겟: 44×44pt
- 색상만으로 상태를 전달하지 않음 (아이콘 또는 텍스트 병용)

## 8. UX 원칙 및 톤앤매너

### UX 원칙

| 원칙 | 설명 |
|------|------|
| 선제적(Proactive) | 사용자가 묻기 전에 필요한 정보를 먼저 제공 |
| 간결함(Conciseness) | 핵심만 전달, 불필요한 장황함 제거 |
| 실행 가능성(Actionability) | 모든 AI 응답에는 맥락별 퀵 액션 칩 제공 |
| 투명성(Transparency) | AI가 어떤 데이터를 바탕으로 판단했는지 이해 가능하게 표시 |
| 부담 없음(Low Friction) | 사용자에게 추가 설정이나 입력을 최소화 |

### 톤앤매너 기본값

- 기본 말투: 친근하고 자연스러운 존댓말 (선택한 대화 스타일에 따라 변형)
- 문장 길이: 1~2문장으로 핵심 전달
- 부정적 표현 지양: "못 했어요" 대신 "아직 진행 중이에요"
- 사용자 이름 호칭: Flow 내 "사용자 이름" 필드 값 사용

### AI 비서 페르소나 (선택 기반)

- 이름: 선택한 캐릭터 이름 (플로키 / 펭귀니 / 거북이)
- 기본 캐릭터: 플로키 (호기심 많은 바다 탐험가)
- 대화 스타일: 선택한 스타일에 따라 응답 톤 조정
- 인사 문구: 시간대별 감성 인사 랜덤 1건 표시 (§5-2 프로필 카드 인사 문구 풀 참조)
- 금지 표현: 과도한 감탄사, 지나친 격려 ("대박이에요!", "너무 잘하셨어요!" 등)
- 권장 표현: "확인해보시면 좋을 것 같아요", "아직 처리 전인 항목이 있어요"
